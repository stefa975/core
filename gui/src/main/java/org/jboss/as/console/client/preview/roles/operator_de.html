Bediener

Die Bediener-Rolle ergänzt die Monitor-Rolle um die Fähigkeit zum Ändern des Laufzeitstatus des Servers. Das bedeutet, dass Bediener den Server neu laden und herunterfahren und außerdem JMS-Ziele anhalten und wiederaufnehmen können. Die Bediener-Rolle eignet sich optimal für Benutzer, die für die physischen oder virtuellen Hosts des Applikationsservers zuständig sind, damit sie sicherstellen können, dass Server bei Bedarf heruntergefahren und ordnungsgemäß neu gestartet werden können. Bediener können die Serverkonfiguration nicht ändern und haben keinen Zugriff auf sensible Daten oder Vorgänge.
