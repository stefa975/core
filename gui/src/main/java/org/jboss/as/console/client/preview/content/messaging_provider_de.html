Messaging-Provider

Ein Messaging-Provider ist eine aktive ActiveMQ-Serverinstanz.

Warteschlangen/Topics

Warteschlangen und Topics erstellen, die von diesem Messaging-Provider verwendet werden.

Verbindungen

Acceptors verwalten, die definieren, wie Verbindungen zum Messaging-Provider hergestellt werden können.

Bridges erstellen, die Nachrichten von einer Quellwarteschlange konsumieren und diese an eine Zieladresse weiterleiten, die sich in der Regel auf einem anderen Server befindet.

Clustering

Definieren Sie Broadcast- und Discovery-Gruppen, die steuern, wie Konnektoren von diesem Provider verarbeitet und verteilt werden.

Verwalten Sie Cluster-Verbindungen zur Lastenverteilung zwischen Knoten eines Clusters.
